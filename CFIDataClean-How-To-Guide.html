CFI DataClean User Guide

A plain-language guide for cleaning CFI forest inventory data in Microsoft Access.

Developed by the BIA Division of Forestry, Branch of Forest Inventory and Planning.

Last reviewed for app build 1.0.00 on July 8, 2026.

Short version: Pick an Access database, review the Run options, optionally add AI settings or a project manual on the AI guidance tab, click Run, then review the exported Excel workbook. CFI DataClean checks a temporary copy of the database and does not change your original database during a Run.
Build 1.0.00 note: This release is the current modern-UI copy of CFI DataClean. It keeps the cleaner v3 window and includes recent refinements for Total Height Protocol controls, always-split history export columns, selected-period DBH shrinkage, duplicate height-shrinkage cleanup, clearer run-option grouping, the BIA header credit, front-loaded current/previous period number and error-count columns, a TotalHeight tab layout that puts previous/current height comparison columns directly after RuleName, a tree IDBH tab layout that separates green IDBH jump checks from red IDBH shrinkage checks, field-tab color zones, detailed performance logging, the PeriodScope export column for filtering current-period, previous-period, other-period, and all/no-period rows, legacy TotalHeight support for old problem codes and live TreeClass/TreeStatus values, clearer regen square-count rows decoded from RegenMeasKey, stronger regen required-species/stem-count checks, built-in regen MinorPlot required checks, fixed regen SpeciesCode identity columns in report exports, plot/tree/regen rule-coverage diagnostics for likely setup gaps, stronger selected-period filtering for regen rows that use ProjectPeriodID, corrected regen StemCount max handling so the configured Max regen stems value is used in SQL and export wording, and startup run-log lines that print the configured range limits for easier troubleshooting.

1. What CFI DataClean Is For

CFI DataClean helps review forest inventory data for likely data-entry mistakes and internal consistency issues. It is designed for CFI-style Microsoft Access databases.

CFI DataClean can help find issues such as:

  • Missing, zero, negative, or unusually large DBH, height, and stem-count values.
  • Invalid code values based on the database setup tables.
  • TreeHistory transitions that do not make sense across measurement periods.
  • Ingrowth mistakes, such as using TreeHistory 10 on install plots.
  • Plot data recorded on plots coded as missing, not measured, dropped off reservation, or dropped other, except for allowed preload/remark fields.
  • Tree data recorded on plots coded as missing, not measured, dropped off reservation, or dropped other.
  • Regen data recorded on plots coded as missing, not measured, dropped off reservation, or dropped other.
  • TreeClass or TreeStatus values that may not match TreeHistory.
  • Plot, tree, regen, and custom-measurement tables that do not line up.
  • Missing required plot entries, tree species, crown fields, radial increment, and regen fields.
Important: CFI DataClean points out records that need review. It does not decide every correction for you. Use the exported workbook, field records, project manual, and Access verification SQL to confirm changes before updating the database. Run the main CFI DataClean cleaning review after project codes have been crosswalked to the current standard. During a crosswalk, ignore cleaning-style findings that are caused by old standard codes changing meaning; treat those as crosswalk review items, not final data-entry errors.

3. Before You Start

Before running CFI DataClean, make sure you have:

  • A CFI Access database file, usually ending in .mdb or .accdb.
  • Microsoft Access or the 32-bit Access Database Engine installed.
  • A project manual, if you want the optional AI features to use project-specific guidance.
  • The correct woodland species codes, if woodland species should be excluded from shrinking DBH checks.
  • Any project-specific minor-plot rules for tree height or regen collection.

4. Opening the App

  1. If CFI DataClean was sent as a ZIP file, right-click the ZIP and choose Extract All. Do not run CFI DataClean from inside the compressed-folder preview.
  2. Open the CFI DataClean project folder.
  3. Double-click CFI DataClean.cmd.
  4. Wait for the loading splash to close and the CFI DataClean window to open.

If the app opens briefly and then closes, open _CFIDataClean_AppFiles and double-click CFIDataClean-debug.cmd. The debug launcher keeps the command window visible so the startup message can be read.

5. Choosing a Database

  1. Click Browse beside Access database.
  2. Choose the Access database you want to review.
  3. CFI DataClean connects automatically after you choose the file.

There is no separate Connect button. Once the database path is selected, CFI DataClean tries to connect and read the database setup.

Safety note: When you click Run, CFI DataClean checks a temporary copy of the database. Your selected Access database is not changed by the Run workflow.

6. Adding a Project Manual for AI

The project manual is optional and is used only by the AI features. Uploading a manual does not change CFI DataClean's built-in cleaning checks, does not decide which records are flagged, and does not update the Access database.

When AI is enabled, CFI DataClean can send relevant manual excerpts to the model so AI chat answers and AIMessage export guidance can be more project-specific.

  1. Open the AI guidance tab.
  2. Click Browse beside Project manual.
  3. Choose a manual or guide file.
  4. CFI DataClean reads the manual text when possible.

Supported manual formats include .pdf, .docx, .xlsx, .xlsm, .xls, .txt, .md, .csv, and .json.

PDF manuals work best when the text is selectable. Scanned image-only PDFs may need OCR before CFI DataClean can read them well.

7. Understanding Run Options

The Run options tell CFI DataClean how strict to be for this project. You should review these before clicking Run.

Measurement period options

Option Plain-language meaning
Limit normal checks to measurement period(s) Keep this on when you only want most checks to look at the current and previous measurement periods. TreeHistory transition checks still review all periods.
Single measurement project Use this for a first-measurement project. CFI DataClean fixes the current period at 1 and ignores the previous-period box.
Current measurement period # The period being reviewed now. CFI DataClean fills this from the highest PeriodNumber it finds.
Previous measurement period # The earlier period used for comparison. CFI DataClean fills this from the next lower PeriodNumber it finds.

Range and jump options

Option Default What it controls
Max IDBH 500 Largest normal tree DBH value. The box allows up to 3500 for very large-tree projects.
Max height 150 Largest normal tree height value.
IDBH jump 100 Flags a large DBH increase between measurement periods.
Height jump 20 Flags a large height increase between measurement periods.
Max regen stems 100 Largest normal regen stem count.

Tree DBH requirement options

  • Require DBH for new mortality (TreeHistory 2/3): Turn this on only when the project requires DBH for new mortality trees.
  • Require DBH for old mortality (TreeHistory 7): Turn this on only when the project requires DBH for old mortality trees.

Total Height Protocol options

Some projects measure height on every live tree, while other projects measure only a subsample. Use the Total Height Protocol option before clicking Run so CFI DataClean does not over-report missing heights.

  • Require height for new mortality (TreeHistory 2/3): Turn this on only when the project requires height for new mortality trees.
  • Require height for old mortality (TreeHistory 7): Turn this on only when the project requires height for old mortality trees.
  • Minor plots where tree height is required: Use this when tree height is required only on certain minor plots. Leave blank when the rule should apply to all minor plots.
Protocol When to use it
Normal required-height rules Use this for the standard CFI DataClean behavior. Blank live-tree heights are checked using the selected TreeHistory and minor-plot rules.
100% live-tree heights Use this when every live tree with TreeHistory 0, 5, or 10 requires total height, unless a selected problem code such as 123, 127, or 128, or old legacy code 72, 74, or 75, means height should be blank.
Subsample by species and 2-inch IDBH class Use this when the project only requires a minimum number of heights per plot, species, and 2-inch IDBH class. CFI DataClean reports grouped subsample shortages instead of flagging every blank live-tree height.

For subsample mode, the default minimum count is 2 and the minimum eligible IDBH is 50 for 5.0 inches. The Require all heights at/above IDBH checkbox is off by default. Turn it on only when a project requires every eligible tree above a project-specific IDBH threshold, such as 170 for 17.0 inches. These boxes use recorded IDBH units.

CFI DataClean assumes TreeNumber starts at north and increases clockwise. In subsample mode, it can flag a group when enough heights were recorded overall, but one of the first required eligible TreeNumbers is blank while a later TreeNumber has height.

Use Rare species requiring 100% live-tree height for species that always require height when TreeHistory is 0, 5, or 10. Enter numeric SpeciesCode values such as 122, 202. Selected problem-code no-height exceptions still apply.

For older databases that have not been crosswalked yet, CFI DataClean also treats TreeClass 1, 2, 3, 4, and 9, and TreeStatus 1, 2, and 3, as live-tree status for TotalHeight checks.

Problem-code height options

CFI DataClean can flag entered height values for timber trees when certain problem codes usually mean height should be blank:

  • 127 - timber broken or missing top.
  • 128 - timber dead top.
  • 123 - timber lean greater than 15 degrees.
  • Old legacy codes are honored automatically: 74 is broken top, 75 is dead top, and 72 is lean greater than 15 degrees.

For these selected timber problem-code cases, blank height is treated as correct, including new ingrowth trees with TreeHistory 10. Woodland species entered in the woodland species box are skipped for these problem-code height rules.

Woodland species box

Use Woodland species to exclude from shrinking diameters when woodland species are allowed to shrink in DBH and should not be flagged by shrinking DBH checks.

Woodland trees are different from timber trees because they may have several measured stems or limbs. A lower DBH or EDRC in a later period can be valid when a stem or limb was cut, died, broke off, or no longer qualifies as a measured stem. In that situation, the measured stem count and overall diameter summary can go down without being a data-entry error.

Enter numeric species codes separated by commas, spaces, semicolons, or pipes.

Example: 65, 66, 68

Regen minor-plot rules

Some projects collect seedlings and saplings on different minor plots. CFI DataClean lets you enter allowed minor plots for six regen groups:

  • Timber seedlings, IDBH 0.
  • Timber saplings, IDBH 20.
  • Timber saplings, IDBH 40.
  • Woodland seedlings, IDBH 0.
  • Woodland saplings, IDBH 20.
  • Woodland saplings, IDBH 40.

CFI DataClean always flags blank Regen MinorPlot when SpeciesCode is greater than 0. Leave a category box blank only when that category should not have an allowed-value list enforced. When a category box has values, CFI DataClean also flags nonblank MinorPlot values that are not in that list.

If matching regen rows exist but the setup does not give CFI DataClean enough information to verify the rule, CFI DataClean adds a Rule coverage diagnostic row to the export. Examples include no regen MinorPlot allowed lists entered, a blank allowed-list box for a regen group that has matching records, woodland MinorPlot rules entered without woodland species codes, or SpeciesCode marked inactive in AppColumns. Treat these as setup warnings to review, not as direct value edits.

Split DBH, height, and TreeHistory history into columns

CFI DataClean always splits DBH, TotalHeight, and TreeHistory history into separate export columns when those values are available. This makes history comparisons easier to read in Excel. Height comparison rows include a HeightChange column, where positive means the current or later height is taller and negative means it is shorter.

8. Optional AI Guidance

AI is optional. CFI DataClean can run without AI, and the main cleaning rules still work.

The AI controls are on the AI guidance tab so they stay out of the way during normal Run setup. Use AI when you want extra project-specific wording in the workbook or want to ask questions about the connected database.

The project manual upload is also on the AI guidance tab because the manual is AI context only. The manual can help AI write better AIMessage guidance, but it does not change the core CFI DataClean rules.

API keys and Azure AI access

API keys are granted through the BIA AI point of contact. You do not need an API key to run CFI DataClean. If you do not have one, you can still run the built-in cleaning checks and export the Excel workbook.

API keys are accessed through the BIA Azure AI endpoint in Microsoft Azure AI Foundry at https://ai.azure.com/.

  5. Click the model you want to use from the list of available models.
  6. In CFI DataClean, replace the model name with that selected model or deployment name.

9. Running the Checks

  1. Select the Access database.
  2. Optional: open the AI guidance tab and select a project manual if you plan to use AI chat or AI export guidance.
  3. Review the Run options.
  4. Optional: open the AI guidance tab, turn on AI guidance, and test the connection.
  5. Click Run.
  6. Read the confirmation message and click Continue if everything looks right.
  7. Watch the status text, progress bar, elapsed time, and window title.
  8. When the run finishes, open the Excel workbook when prompted.

During a run, CFI DataClean disables controls that could change the run. This prevents accidental changes while the app is checking the temporary database copy.

If you need to stop a run, click Cancel run. CFI DataClean cancels between major steps. It may wait for the current Access query or AI request to finish first.

CFI DataClean also creates a matching text run log beside the workbook, using a name like Mescalero_CFIDataCleanCleaning_20260604_1200_RunLog.txt. If the run fails before the Excel workbook is created, check that text log for the failure message, major progress step, and troubleshooting details. The log also records source database size, temporary-drive free space, major step timings, and slow Access SQL previews so slow runs can be diagnosed and improved.

10. What the Excel Workbook Contains

CFI DataClean saves a separate Excel workbook. The workbook is the main thing you review after a run.

Numeric-looking values in the workbook are written as Excel numbers where it is safe to do so, so they sort, filter, and calculate like numbers. Leading-zero codes such as 020 stay as text so the code is not changed.

Detail report tabs start with the most-used review columns: rule name, current-period number, current-period error count, previous-period number, previous-period error count, recorded value, verification SQL, and key plot/tree context. The period error-count columns show 1 when that finding belongs to the selected current or previous period and 0 otherwise. The TotalHeight tab is arranged differently for easier review: after rule name it shows previous period number, previous total height, current period number, current total height, and height change before the recorded value and verification SQL. The tree IDBH tab is also arranged for DBH review: IDBH jump checks use previous period number, previous IDBH, current period number, current IDBH, and IDBHJump, while IDBH shrinkage checks use earlier period number, earlier IDBH, later period number, later IDBH, and shrinkage. IDBH jump cells are shaded green; IDBH shrinkage cells are shaded red. Regen IDBH stays separate because regen IDBH is a class-code check, not a tree DBH shrinkage check. PeriodScope still appears beside PeriodNumber so reviewers can filter rows by current period, previous period, other periods, or all/no-period checks. Field tabs use lightweight color zones: blue for review/action columns, yellow for the recorded value, green for supporting context, and gray for extra audit details at the end.

Workbook part What it means
Summary High-level list of worksheet names, total error counts, current-period error counts, previous-period error counts, other/all-period error counts, tables, and rules.
Database Square Always directly after Summary. Shows whether plot, tree, regen, and custom-measurement counts line up for every measurement period in the database, even when normal cleaning checks are limited to selected periods. Regen does not have a separate parent table like Trees, so CFI DataClean shows a RegenMeasurements count by RegenMeasKey row and then compares RegenCustomMeasurements to it using the period decoded from RegenMeasKey.
Missing Trees Always directly after Database Square. Shows missing tree measurement rows if found, or an OK row if none are found.
Run Data Settings used for the run, source database path, export path, text run-log path, thresholds, period scope, manual, AI settings used, runtime, audit counts, and the top slow timed steps. The API key is never included.
Field tabs One tab per reviewed field, such as IDBH, TotalHeight, StemCount, SpeciesCode, TreeHistory, Regen IDBH, or custom fields.

Tab colors

  • Red tabs: contain findings that need cleaning review.
  • Green tabs: were reviewed and passed, meaning no data-entry errors were found for that active field.
  • Database Square and Missing Trees: always appear directly after Summary. They are red when they contain errors and green when no errors are found.
  • Run Data: follows those two project-wide check tabs.

Important columns

Column How to use it
RuleName The type of issue CFI DataClean found.
RecordLabel A readable label for the row, such as plot/tree/period information.
Recorded...Value The value or comparison that triggered the finding. This column is highlighted yellow.
Verification SQL A SELECT query you can copy into Access to verify the finding yourself.
AIMessage Project-specific guidance when AI export guidance was turned on.
TreeRemarks, PlotRemarks, RegenRemarks Remarks from the database. These show even when blank so you know whether a remark was recorded.

Summary period counts

The Summary tab separates total errors by measurement period when CFI DataClean can identify the period for a row.

  • CurrentPeriodNumber is the selected current measurement period.
  • CurrentPeriodErrorCount is how many errors on that worksheet belong to the current period.
  • PreviousPeriodNumber is the selected previous measurement period.
  • PreviousPeriodErrorCount is how many errors on that worksheet belong to the previous period.
  • OtherOrAllPeriodErrorCount is used for all-period checks, rows without a clear period, and rows outside the selected current/previous periods.
  • CurrentPeriodNumber, CurrentPeriodErrorCount, PreviousPeriodNumber, PreviousPeriodErrorCount, and PeriodScope appear on the detail report tabs so reviewers can filter individual rows the same way the Summary tab counts them.
  • Field tab color zones use blue for review/action columns, yellow for recorded values, green for supporting plot/tree/period context, and gray for extra audit detail columns.
  • Summary count columns are color-coded: total and other/all-period counts are yellow, current-period columns are blue, and previous-period columns are green. Nonzero count values are bold, and plot/tree/regen summary sections are separated with a thick line.

11. How to Review Findings

  1. Start with the red tabs at the front of the workbook.
  2. Read the plain-language Checked note at the top of the tab.
  3. Find the yellow recorded-value column.
  4. Use PlotNumber, TreeNumber, SpeciesCode, MinorPlot, PeriodNumber, TreeKey, or RegenMeasKey to find the record in the database or field notes.
  5. Read TreeRemarks, PlotRemarks, or RegenRemarks before deciding whether a value is wrong.
  6. Copy the VerificationSql into Access if you want to reproduce the finding.
  7. Make database updates only after the field record or project guidance confirms the correction.

12. Important Cleaning Rules in Plain English

AppColumns controls what gets reviewed

CFI DataClean reviews fields marked Active in the database AppColumns table. If a data item is not active, CFI DataClean usually skips it for data-entry cleaning. Some items, such as per-acre expansion fields, CalcSiteIndex, and GMP, are skipped even if active because they are not cleaning items.

CFI DataClean also adds Rule coverage diagnostic rows when matching records exist but a setup gap may prevent the check from fully running. These are not direct data edits. They are warnings such as: PlotStatus is missing or inactive, a required plot field is missing or inactive, TreeHistory is inactive, a tree species join is unavailable, a crown or radial increment field is missing or inactive, a needed join key such as PlotID or PeriodNumber is missing, or a regen minor-plot rule needs more setup.

PlotStatus progression

  • CFI DataClean checks PlotStatus across all periods when the PlotStatus field is active in AppColumns.
  • Before a plot is installed, earlier periods should be blank or null.
  • The first nonblank PlotStatus for a plot should be 2, which means initial install.
  • After a plot has received PlotStatus 2, later periods should be 1, 4, 5, 6, or 7. Later periods should not go back to blank.
  • A PlotStatus 5 not measured value before any initial install code is flagged for review.

Tree DBH

  • TreeHistory blank, 1, 4, 6, 8, and 9 can have blank DBH.
  • TreeHistory 1, 4, and 8 should not have DBH recorded.
  • TreeHistory 2, 3, and 7 can be checked for required DBH only if the matching Run option is turned on.

Plot data on not-measured or dropped plots

  • If PlotStatus is 4 missing plot, 5 not measured, 6 dropped - off reservation, or 7 dropped - other, most plot measurement and plot custom measurement data should not be recorded for that plot period.
  • CFI DataClean checks active plot fields and flags recorded values that should not be there.
  • Plot remarks are allowed because dropped plots often need an explanation.
  • UTM coordinates, management unit, and FLCCommercial are allowed because they are usually preloaded or still useful for locating/classifying the plot.
  • Key fields such as PlotID, PlotKey, PlotMeasKey, MeasurementID, and PeriodNumber are not treated as plot data for this rule.

Tree height

  • TreeHistory blank, 1, 4, 6, 8, and 9 can have blank height.
  • Height can be required for new or old mortality only when the matching Run option is turned on.
  • Height requirements can be limited to entered minor plots.
  • A shorter later height is allowed when a previously live tree, TreeHistory 0, 5, or 10, becomes a mortality tree whose height is required by the selected Run options, TreeHistory 2, 3, or 7. Dead trees can lose height through breakage.

Problem and severity codes

  • Problem and severity fields should agree. If a problem code is entered, the matching severity should be entered too.
  • If TreeHistory is 2 or 3 and Problem1 is recorded, Severity1 must be code 3.
  • Blank, Null, and 0 count as no Problem1. Values greater than 0 count as an entered Problem1.

Tree data on not-measured or dropped plots

  • If the matching plot status is 4 missing plot, 5 not measured, 6 dropped - off reservation, or 7 dropped - other, tree measurement and tree custom measurement data should not be recorded for that plot period.
  • CFI DataClean checks active tree fields and flags recorded values such as TotalHeight, IDBH, TreeHistory, crown fields, problem codes, radial increment, and active tree custom fields.
  • Key fields such as TreeID, TreeKey, TreeMeasKey, MeasurementID, and PeriodNumber are not treated as tree data for this rule.

Regen data on not-measured or dropped plots

  • If the matching plot status is 4 missing plot, 5 not measured, 6 dropped - off reservation, or 7 dropped - other, regen measurement and regen custom measurement data should not be recorded for that plot period.
  • CFI DataClean checks active regen fields and flags recorded values such as MinorPlot, SpeciesCode, IDBH, StemCount, and active regen custom fields.
  • The exported row includes the matching PlotStatus code so reviewers can see why the regen value was flagged.
  • Regen remarks are allowed because they may explain why the plot was not measured or was dropped.
  • Regen records are linked by RegenMeasKey and plot/period information, not by TreeKey.

Ingrowth

  • New live trees on install plots with PlotStatus 2 should normally be TreeHistory 0.
  • TreeHistory 10 is for new ingrowth trees on remeasurement plots.
  • TreeHistory 10 in the first project measurement period is flagged because first-period trees should normally be 0.
  • TreeHistory 10 is flagged when the same tree has an earlier nonblank TreeHistory value. Earlier blank TreeHistory rows do not count as prior history.
  • A first-recorded live tree coded 0 on a PlotStatus 1 remeasurement plot after the first project period is flagged as possible missed ingrowth.

The ingrowth check joins tree and plot measurements by both plot and measurement period before filtering. This helps avoid oversized Access temporary joins on large projects.

Regen

  • Regen rows with RegenMeasKey, IDBH, StemCount, or MinorPlot data require SpeciesCode.
  • Regen rows with SpeciesCode greater than 0 require IDBH, StemCount, and MinorPlot.
  • Entered regen StemCount values are checked against the max regen stem count even when SpeciesCode is blank.
  • When selected-period filtering is on, CFI DataClean filters regen rows by PeriodNumber if present, then by ProjectPeriodID joined to ProjectMeasurementPeriods, and only falls back to decoding period from RegenMeasKey.
  • Valid regen IDBH class codes are 0, 20, and 40.
  • Regen rows use RegenMeasKey, not TreeKey.
  • CFI DataClean adds Rule coverage diagnostic rows when setup may prevent full review, such as inactive SpeciesCode, missing prerequisite fields, blank matching MinorPlot allowed-list groups, or missing woodland species codes for woodland regen rules.

TreeClass and TreeStatus crosswalk checks

When TreeClass or TreeStatus fields exist and are active in AppColumns, CFI DataClean compares them with TreeHistory using the supplied crosswalk logic. Clear mismatches are flagged. Conditional cases are sent to review instead of guessed. TreeClass 1, 2, 3, 4, and 9, and TreeStatus 1, 2, and 3, are treated as live and should generally align with TreeHistory 0, 5, or 10.

Recommended workflow: complete the crosswalk first, then run CFI DataClean as the main cleaning review. A light pre-crosswalk review is still useful for obvious structural problems, such as missing keys, duplicate records, broken relationships, or impossible numeric values. It is not the right time to make final calls on fields whose standard codes changed.

During the crosswalk, ignore cleaning-style findings that only exist because old project codes have not been converted yet. After the crosswalk is finished, rerun CFI DataClean and use the exported findings as the main cleaning workbook. If a finding still appears after crosswalking, verify it against the project manual, field records, or project-specific code-change notes before editing values.

The supplied crosswalk reference workbook is included with CFI DataClean: TreeClass_Status_History_Crosswalk.xlsx. If this guide is opened in Microsoft Word, use Ctrl+click on the link or open the workbook from _CFIDataClean_AppFiles\References.

13. Clearing the Screen

Click Clear all beside Run when you want to remove the current database, uploaded project manual, woodland species-code list, and minor-plot rules from the app window.

Clear all does not delete any database or workbook files. It only clears the current app selections.

14. Troubleshooting

Problem What to try
The app opens and closes immediately. Open _CFIDataClean_AppFiles, run CFIDataClean-debug.cmd, and read the visible message. The most common issue is missing Access/ACE database drivers. If the message says app files are missing, extract the ZIP first and run CFI DataClean from the extracted folder.
The debug window says the PowerShell script is not digitally signed. Use the latest CFI DataClean launcher. It starts the app with a per-run PowerShell bypass so the user's computer-wide policy is not changed. If agency Group Policy still blocks it, IT will need to code-sign _CFIDataClean_AppFiles\ForestInventoryCleaner.ps1 or deploy CFI DataClean through an approved software channel.
The app seems frozen. Look at the progress text, progress bar, elapsed time, and window title. Large Access queries and AI requests can take time. Use Cancel run if needed.
A run failed and there is no Excel workbook. Look for the matching _RunLog.txt file beside the workbook location you selected. The popup and on-screen log should also show this path. If Access failed during a write query, the log includes a shortened SQL preview.
No AI message appears in the workbook. AI may be off, not fully set up, or Azure may have rejected a guidance request. The built-in findings still run without AI.
AI test says the endpoint is wrong. Use only the real HTTPS endpoint. Do not paste a DOI/network prefix in front of it.
A field was not reviewed. Check whether that field is active in AppColumns and whether valid codes are listed in AppColumnCodes when needed.
A custom measurement code was not checked. Make sure the custom measurement field and code values are set up in AppColumns and AppColumnCodes.

15. Glossary

Term Meaning
Access database The .mdb or .accdb file that stores the project data.
AppColumns A setup table that tells CFI DataClean which fields are active and should be reviewed.
AppColumnCodes A setup table that lists valid code values for coded fields.
IDBH Tree diameter value used by the project. For regen, IDBH is a class code: 0, 20, or 40.
TreeHistory A code describing the tree's status/history across measurement periods.
RegenMeasKey A key used for regen measurement records. It includes plot, period, minor plot, species, and IDBH class information.
VerificationSql An Access SELECT query provided by CFI DataClean so you can verify a finding yourself.

This guide should be reviewed whenever CFI DataClean is updated. If the app adds, removes, or renames a button, Run option, export column, report tab, cleaning rule, AI behavior, or launcher file, update this guide before sharing the new build.

Shared release folders should include the app build number in the folder name, such as CFI DataClean v1.0.00. When a new app build is released, update the app version in _CFIDataClean_AppFiles\ForestInventoryCleaner.ps1, update this guide, and rename the shared folder so the folder name matches the build.
